### BPS が空になるケース（重要）

- **IFRS 採用銘柄の中間決算（2Q など）では `BPS` が空**
- `BPS` には非連結(`NCBPS`)バリアントもあるが同様に空のことがある
- **代替計算**: `BPS = Eq / (ShOutFY - TrShFY)` で算出可能
  - `Eq`（自己資本）と株式数は提供されているため計算は可能

## 実装への影響

### スキーマ変更

| 変更点 | 内容 |
|--------|------|
| `daily_prices.va` 追加推奨 | 売買代金（流動性フィルターに活用） |
| `financial_summary.bps` | NULL 許容で実装（IFRS 中間では空） |
| `financial_summary.bps` 計算ロジック | `BPS` 空の場合は `Eq / (ShOutFY - TrShFY)` で代替 |

### PBR 計算方針

```
BPS =
  CASE
    WHEN fins_summary.BPS IS NOT NULL AND fins_summary.BPS != '' THEN fins_summary.BPS
    ELSE fins_summary.Eq / (fins_summary.ShOutFY - fins_summary.TrShFY)
  END

PBR = 最新終値 (daily_prices.adj_close) / BPS
```

### データ鮮度

- 無料プランでは **最新データが約 3 ヶ月前**のため、画面に「データは〜までの情報です」と表示する
- 初期同期時は `from=2023-11-29&to=2025-11-29` の全期間を取得する

### 銘柄コード形式

- API レスポンスの `Code` は **5 桁**（例: `72030`）
- DB の `stock_master.code` カラムは 5 桁で保持する
- URL パラメータは 4 桁（`/stock/7203`）で受け取り、クエリ時に `LIKE '7203%'` か `= '72030'` に変換する
